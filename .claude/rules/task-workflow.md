# 작업 흐름 규칙

## 세션 시작 가드 (최우선)

**모든 작업보다 먼저** 아래 3가지를 확인한다. 실패 시 작업을 시작하지 않는다.

1. `CLAUDE.md`가 없으면 → 사용자에게 안내: `⚠️ CLAUDE.md가 없습니다. /init을 먼저 실행하세요.`
2. `CLAUDE.md`에 `⚠️ 초기화 미완료` 마커가 있는지 확인
   - 있으면 → 사용자에게 안내: `⚠️ 프로젝트 초기화가 완료되지 않았습니다. /sync-claude를 먼저 실행하세요.`
   - 사용자가 명시적으로 "무시하고 진행해"라고 하지 않는 한 작업을 시작하지 않는다
3. **기술 스택 정합성 검증** — `package.json`의 주요 의존성(`dependencies` 키)을 빠르게 읽고, `CLAUDE.md` 기술 스택 섹션과 대조한다
   - 주요 프레임워크 불일치 감지 시 (예: `next`가 추가되었는데 CLAUDE.md에 없음, 또는 `vite`가 삭제되었는데 CLAUDE.md에 남아 있음):
     ```
     ⚠️ 기술 스택 변경 감지
     - package.json: next 15.x 추가 / vite 삭제
     - CLAUDE.md: 여전히 "React 19 + Vite 6"
     CLAUDE.md를 갱신하고 /sync-claude를 실행하세요.
     ```
   - 사소한 패키지 추가/삭제는 무시한다 (프레임워크급 변경만 감지)

## 작업 모드

| 모드         | 트리거                              | 동작                    |
| ------------ | ----------------------------------- | ----------------------- |
| 즉시 실행    | 요청이 명확하고 범위가 작을 때      | 바로 구현               |
| 계획 후 실행 | 범위가 크거나 아키텍처 변경 포함 시 | 계획 공유 → 승인 → 구현 |
| 질문 우선    | 요구사항이 모호할 때                | 핵심 질문 1~2개 → 구현  |

- **Plan 모드** (가벼운 작업): 파일 1~2개 수정, 단일 기능 변경, 버그 수정, 리팩토링
- **`/plan-feature` 스킬** (복잡한 작업): 새 기능 추가, 3개 이상 파일 변경 예상, PRD/ROADMAP 갱신 필요, 아키텍처 변경 수반
- Plan 모드 진행 중 작업이 예상보다 커지면 중단하고 `/plan-feature` 전환을 제안한다
