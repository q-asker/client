# 작업 규칙

## 세션 시작 가드 (최우선)

**모든 작업보다 먼저** 아래 3가지를 확인한다. 실패 시 작업을 시작하지 않는다.

1. `CLAUDE.md`가 없으면 → 사용자에게 안내: `⚠️ CLAUDE.md가 없습니다. /init을 먼저 실행하세요.`
2. **기술 스택 정합성 검증** — `package.json`의 주요 의존성(`dependencies` 키)을 빠르게 읽고, `CLAUDE.md` 기술 스택 섹션과 대조한다
3. **rules 정합성 검증 (카나리아)** — 이미 로딩된 `.claude/rules/` 파일들의 내용이 CLAUDE.md 기술 스택·아키텍처와 일치하는지 확인한다
   - 명백한 불일치 감지 시 (예: rules에 "Spring Boot" 언급, CLAUDE.md는 "React") → `/init` 실행 제안
   - 프레임워크/아키텍처급 불일치만 감지한다 (세부 라이브러리 차이는 무시)
   - rules 불일치는 agents, commands 등 `.claude/` 전체가 다른 프로젝트 것일 가능성을 의미하므로, `/init`으로 전체 동기화를 유도한다
   - 주요 프레임워크 불일치 감지 시 → CLAUDE.md의 기술 스택 섹션을 직접 수정하고, `/init`을 실행한다:

     ```
     ⚠️ 기술 스택 변경 감지
     - package.json: next 15.x 추가 / vite 삭제
     - CLAUDE.md: 여전히 "React 19 + Vite 6"

     CLAUDE.md 기술 스택 섹션에서 "React 19 + Vite 6" → "React 19 + Next.js 15"로
     변경해도 될까요? 승인하시면 수정 후 /init을 실행하겠습니다.
     ```

   - 사소한 패키지 추가/삭제는 무시한다 (프레임워크급 변경만 감지)

## 작업 모드

| 모드         | 트리거                              | 동작                    |
| ------------ | ----------------------------------- | ----------------------- |
| 즉시 실행    | 요청이 명확하고 범위가 작을 때      | 바로 구현               |
| 계획 후 실행 | 범위가 크거나 아키텍처 변경 포함 시 | 계획 공유 → 승인 → 구현 |
| 질문 우선    | 요구사항이 모호할 때                | 핵심 질문 1~2개 → 구현  |

- **Plan 모드** (가벼운 작업): 파일 1~2개 수정, 단일 기능 변경, 버그 수정, 리팩토링
- **`/plan-feature` 스킬** (복잡한 작업): 새 기능 추가, 3개 이상 파일 변경 예상, PRD/ROADMAP 갱신 필요, 아키텍처 변경 수반
- Plan 모드 진행 중 작업이 예상보다 커지면 중단하고 `/plan-feature` 전환을 제안한다

## 에이전트

| 에이전트              | 위치                   | 용도                         |
| --------------------- | ---------------------- | ---------------------------- |
| `code-reviewer`       | `.claude/agents/dev/`  | 코드 구현 완료 후 품질 리뷰  |
| `development-planner` | `.claude/agents/dev/`  | ROADMAP 생성/갱신, Task 관리 |
| `prd-generator`       | `.claude/agents/docs/` | PRD 생성/갱신                |

## 스킬

| 스킬              | 트리거 조건                           |
| ----------------- | ------------------------------------- |
| `/commit`         | Git 커밋 생성 시                      |
| `/pr`             | PR 작성 시                            |
| `/plan-feature`   | 새 기능/대규모 작업 계획 시           |
| `/init`           | 프로젝트 초기화 및 .claude/ 동기화 시 |
| `/history-logger` | 대화 히스토리 Notion 기록 시          |
| `/cs-logger`      | CS 개념 Notion 기록 시                |

## 위임 원칙

- 스킬이 존재하는 작업은 반드시 해당 스킬을 사용한다
- 스킬 없이 처리 가능한 작업은 직접 수행한다
- 스킬 실행 실패 시 사용자에게 알리고 대안을 제시한다

## 작업 원칙

- 요청 범위를 벗어난 파일은 수정하지 않는다
- 리팩토링이 필요하면 별도 제안으로 분리한다
- `.env` 파일을 Git에 커밋하지 않는다
- 외부 라이브러리 추가 시 CLAUDE.md 기술 스택 섹션 갱신
- `npm run build`가 항상 성공하는 상태를 유지한다
- `npm run lint`에서 에러가 없는 상태를 유지한다
