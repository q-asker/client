# 빈칸 채우기(Fill-in-the-Blank) 설계 컨텍스트

> **이 파일은 빈칸 채우기 프롬프트 평가 시에만 참조합니다.**
> 공통 설계 컨텍스트(`common.md`)와 함께 사용하세요.

---

## 1. 형성 평가 단일 모드

빈칸 채우기는 **형성 평가 전용**으로 설계되었습니다. 분기 로직이 없습니다.

| 설계 결정   | 내용                                                              |
| ----------- | ----------------------------------------------------------------- |
| 평가 목적   | 형성 평가 — 단일 개념(용어/정의/명칭/메서드명)의 정확한 회상·매칭 |
| 인지 수준   | Bloom's Taxonomy '기억(Remember)' 단일 수준                       |
| 예상 정답률 | 70~85% — 핵심 정의 용어(80~85%), 부수적 명칭(70~75%)              |

**평가 시 주의**: `평가 목적` placeholder나 형성/총괄/진단 분기 로직을 제안하거나 도입하는 것은 금지입니다.

## 2. 고정값 요소

아래 요소는 설계자가 확정한 고정값이며, 변경/파라미터화를 제안하지 마세요:

- **빈칸 규칙**: title에 `"_______"` 정확히 1개, 빈칸에 정답만 유일하게 적합
- **선택지 수**: 4개 고정 (정답 1 + 오답 3)
- **선택지 균형**: 품사·형태·글자 수(평균 대비 ±20%) 통일, 코드/약어는 구조적 유사성 기준
- **오답 전략**: 3유형(유사 개념형·혼동 유발형·무관 개념형) 중 최소 2유형 이상 포함
- **해설 구조**: 단일 계층 통합 — selections[0].explanation에 [정답] + [오답 1~3] + [학습 포인트] 전체 포함
- **난이도**: Bloom's '기억' 단일 수준 (난이도 배분 없음)
- **대상 학습자**: 대학 학부 수준
- **출제 방향 배분**: 용어→정의 / 정의→용어 각 최소 30% 이상
- **정답 위치**: 프롬프트에서 지시하지 않음 (서버에서 선택지 셔플 수행)

## 3. 출력 구조

JSON 스키마 의존성을 최소화하기 위해, 프롬프트에서는 **3개 핵심 필드**만 명시합니다:

- `title` (빈칸 포함 문제 서술문 — `"_______"` 1개 필수)
- `selections` (선택지 목록 — 각 `content` + `correct` boolean + `explanation` 해당 선택지 해설)
- 정답 선택지의 `explanation`에 전체 해설 통합 (정답 근거 + 오답별 교정 + 학습 포인트)

**MCQ와의 차이**: MCQ는 `content`(문제 본문) + `selections` + `explanation`(문항 전체 해설) 3필드 구조이지만, 빈칸 채우기는 `title`(빈칸 포함 서술문) + `selections` 구조이며, 해설이 정답 선택지의 explanation에 통합됩니다.

**평가 시 주의**: 출력 필드를 세분화하거나 추가 필드를 제안하지 마세요. 스키마는 별도로 관리됩니다.

## 4. 빈칸 채우기 설계 준수 평가 체크리스트

> 아래 체크리스트는 빈칸 채우기 프롬프트 평가 시에만 적용합니다.
> 공통 체크리스트(`common.md` §4)와 함께 사용하세요.

### 형성 평가 모드 준수 → `clarity`, `difficulty_control` 반영

| #   | 체크 항목                                                  | 위반 예시                                                |
| --- | ---------------------------------------------------------- | -------------------------------------------------------- |
| B1  | 평가 목적이 형성 평가 고정인가? placeholder나 분기 없음    | `{평가 목적 삽입 위치}`, `총괄 평가인 경우...` 분기 존재 |
| B2  | 인지 수준이 Bloom's '기억(Remember)' 단일 수준인가?        | 적용/분석 수준 문항 포함                                 |
| B3  | 예상 정답률 70~85% 범위가 명시되고 조정 메커니즘이 있는가? | 정답률 기준 없음, 또는 단서 조정 메커니즘 없음           |
| B4  | 대상 학습자가 "대학 학부 수준"으로 고정인가?               | `{학과/학년 삽입 위치}` placeholder 존재                 |

### 빈칸 규칙 준수 → `clarity`, `answer_quality` 반영

| #   | 체크 항목                                                   | 위반 예시                                    |
| --- | ----------------------------------------------------------- | -------------------------------------------- |
| B5  | title에 `"_______"` 정확히 1개 규칙이 명시되었는가?         | 빈칸 기호 미정의, 복수 빈칸 허용             |
| B6  | 정답 유일성 보장 메커니즘이 있는가? (동의어·유사 표현 대응) | 정답 유일성 규칙 없음, 동의어 대처 방안 없음 |
| B7  | 출제 방향(용어→정의 / 정의→용어) 배분 규칙이 명시되었는가?  | 출제 방향 다양성 규칙 없음, 한 방향에만 편중 |

### 고정값 준수 → `difficulty_control`, `answer_quality` 반영

| #   | 체크 항목                                                                                 | 위반 예시                                                     |
| --- | ----------------------------------------------------------------------------------------- | ------------------------------------------------------------- |
| B8  | 오답 3유형(유사 개념형·혼동 유발형·무관 개념형)이 정의되고 최소 2유형 포함 규칙이 있는가? | 오답 유형이 1개만 정의, 유형 수 제약 없음                     |
| B9  | 오답 매력도 검증("학습자의 오개념은?") 메커니즘이 있는가?                                 | 오답 설계 기준이 표면적 유사성만 고려                         |
| B10 | 선택지 4개 고정 + 품사/형태/글자 수 균형 규칙이 명시되었는가?                             | 선택지 수 가변, 균형 기준 없음                                |
| B11 | 정답 위치 분산/균등 배치를 프롬프트에서 지시하지 **않는가**?                              | "정답 위치를 분산하세요" 등 존재 (서버에서 셔플하므로 불필요) |

### 해설 구조 준수 → `answer_quality`, `model_optimization` 반영

| #   | 체크 항목                                                                      | 위반 예시                                        |
| --- | ------------------------------------------------------------------------------ | ------------------------------------------------ |
| B12 | 해설이 [정답] + [오답 N] + [학습 포인트] 구조를 따르는가?                      | 구조 미정의, 정답 근거만 제시하고 오답 교정 없음 |
| B13 | 오답별 해설에 유형 명시 + 구별 포인트 + 셀프 점검(메타인지 유도)이 포함되는가? | 오답 해설이 단순 "틀림" 수준, 교정 구조 없음     |
| B14 | 해설에 강의노트 출처 직접 인용(`> 근거: [섹션명] > "인용"`) 규칙이 있는가?     | 출처 인용 없이 설명만 제공                       |

### 출력 구조 준수 → `model_optimization` 반영

| #   | 체크 항목                                                                          | 위반 예시                                  |
| --- | ---------------------------------------------------------------------------------- | ------------------------------------------ |
| B15 | 출력 필드를 `title`, `selections` 중심으로 안내하는가?                             | 13개 이상 상세 필드 정의, JSON 스키마 중복 |
| B16 | selections의 구조가 `content` + `correct`(boolean) + `explanation`으로 안내되는가? | 선택지 라벨(A/B/C/D)을 별도 필드로 정의    |
| B17 | title 금지 항목(직접 참조, 선택지 인덱스, 정답 특정, 응용 요구)이 명시되었는가?    | title 금지 사항 없음                       |

## 5. MCQ와의 주요 차이점 (평가 참고)

| 항목           | MCQ(4지선다)                     | 빈칸 채우기                                    |
| -------------- | -------------------------------- | ---------------------------------------------- |
| 인지 수준      | Bloom's 적용~분석 (중/상 균등)   | Bloom's 기억 (단일)                            |
| 평가 목적      | 통합 모드 (형성+총괄+진단)       | 형성 평가 전용                                 |
| 문제 구조      | content (문제 본문)              | title (빈칸 `"_______"` 포함 서술문)           |
| 정답 유일성    | 선택지 중 1개 정답               | 빈칸에 유일하게 적합한 정답 + 동의어 대응 필수 |
| 오답 유형      | 조건 누락형·오개념형·무관 사실형 | 유사 개념형·혼동 유발형·무관 개념형            |
| 해설 계층      | 2계층 (selections별 + 문항 전체) | 단일 계층 (정답 선택지 explanation에 통합)     |
| 난이도 배분    | 중/상 균등                       | 없음 (단일 수준)                               |
| 예상 정답률    | 미지정                           | 70~85% (단서 기반 조절)                        |
| 출제 방향 배분 | 미지정                           | 용어→정의 / 정의→용어 각 30% 이상              |
