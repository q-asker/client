# MCQ(4지선다) 설계 컨텍스트

> **이 파일은 MCQ(4지선다) 프롬프트 평가 시에만 참조합니다.**
> 공통 설계 컨텍스트(`common.md`)와 함께 사용하세요.

---

## 1. 통합 평가 모드

형성·총괄·진단 평가의 장점을 **단일 모드로 통합**한 설계입니다. 분기 로직이 없습니다.

| 설계 결정   | 내용                                                                                      |
| ----------- | ----------------------------------------------------------------------------------------- |
| 평가 목적   | 통합 모드 — 개념 이해 변별 + 오개념 진단·교정 + 학습 촉진                                 |
| 인지 수준   | Bloom's Taxonomy '적용(Apply)' ~ '분석(Analyze)' ~ '평가(Evaluate)' — 기억/이해 수준 금지 |
| 난이도 배분 | 중(적용~분석) / 상(분석+평가) 균등                                                        |

| 출처      | 통합된 장점                                       |
| --------- | ------------------------------------------------- |
| 형성 평가 | scaffolding, 오개념 교정 비중 강화 (4단계 해설)   |
| 총괄 평가 | 변별력, 난이도 분포 엄격 준수 (중/상 균등)        |
| 진단 평가 | 오답 3유형 오개념 진단 + 4단계 해설의 단계적 교정 |

**평가 시 주의**: `평가 목적` placeholder나 형성/총괄/진단 분기 로직을 제안하거나 도입하는 것은 금지입니다.

## 2. 고정값 요소

아래 요소는 설계자가 확정한 고정값이며, 변경/파라미터화를 제안하지 마세요:

- **오답 전략**: 3유형(조건 누락형·오개념형·무관 사실형) 균등 배치
- **해설 구조**: 4단계 2계층 분리 — selections[].explanation(1단계 정답 추론 / 2단계 오개념 교정) + explanation(3단계 자기 점검 + 4단계 심화 학습)
- **난이도 배분**: 중/상 균등 (적용~분석 / 분석+평가)
- **대상 학습자**: 대학 학부 수준
- **선택지 수**: 4개 고정
- **정답 위치**: 프롬프트에서 지시하지 않음 (서버에서 선택지 셔플 수행)

## 3. 출력 구조

JSON 스키마 의존성을 최소화하기 위해, 프롬프트에서는 **3개 핵심 필드**만 명시합니다:

- `content` (문제 본문)
- `selections` (선택지 목록 — 각 `content` + `correct` boolean + `explanation` 해당 선택지 해설)
- `explanation` (문항 전체 해설 — 3단계 자기 점검 + 4단계 심화 학습)

**평가 시 주의**: 출력 필드를 세분화하거나 추가 필드를 제안하지 마세요. 스키마는 별도로 관리됩니다.

## 4. MCQ 설계 준수 평가 체크리스트

> 아래 체크리스트는 MCQ 프롬프트 평가 시에만 적용합니다.
> 공통 체크리스트(`common.md` §4)와 함께 사용하세요.

### 통합 모드 준수 → `clarity`, `difficulty_control` 반영

| #   | 체크 항목                                                                                            | 위반 예시                                                |
| --- | ---------------------------------------------------------------------------------------------------- | -------------------------------------------------------- |
| M1  | 평가 목적이 고정(통합 모드)인가? placeholder나 분기 없음                                             | `{평가 목적 삽입 위치}`, `형성 평가인 경우...` 분기 존재 |
| M2  | 인지 수준이 Bloom's '적용(Apply)' ~ '분석(Analyze)' ~ '평가(Evaluate)' 범위인가? 기억/이해 수준 금지 | 기억(Remember)·이해(Understand) 수준 문항 포함           |
| M3  | 형성·총괄·진단의 장점이 통합되어 반영되었는가?                                                       | scaffolding 없음, 오개념 진단 구조 없음                  |
| M4  | 대상 학습자가 "대학 학부 수준"으로 고정인가?                                                         | `{학과/학년 삽입 위치}` placeholder 존재                 |

### 고정값 준수 → `difficulty_control`, `answer_quality` 반영

| #   | 체크 항목                                                                                      | 위반 예시                                                                          |
| --- | ---------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| M5  | 오답 3유형(조건 누락형·오개념형·무관 사실형) 균등 배치가 명시되었는가?                         | 오답 유형이 2개만 있거나, 유형 비율이 자유                                         |
| M6  | 4단계 해설이 2계층으로 분리되었는가? (selections[].explanation: 1~2단계, explanation: 3~4단계) | 4단계가 단일 텍스트에 통합, 또는 단계 구조 없음                                    |
| M7  | 난이도 중/상 균등 배분이 명시되었는가?                                                         | 난이도 배분이 자유이거나 특정 난이도 편중                                          |
| M8  | 선택지 4개 고정이 명시되었는가?                                                                | 선택지 수가 가변적                                                                 |
| M9  | 정답 위치 분산/균등 배치를 프롬프트에서 지시하지 **않는가**?                                   | "정답 위치를 분산하세요", "A/B/C/D 균등 분배" 등 존재 (서버에서 셔플하므로 불필요) |

### 출력 구조 준수 → `model_optimization` 반영

| #   | 체크 항목                                                                          | 위반 예시                                                 |
| --- | ---------------------------------------------------------------------------------- | --------------------------------------------------------- |
| M10 | 출력 필드를 `content`, `selections`, `explanation` 3개로만 명시하는가?             | 13개 이상 상세 필드 정의, JSON 스키마 중복                |
| M11 | selections의 구조가 `content` + `correct`(boolean) + `explanation`으로 안내되는가? | 선택지 라벨(A/B/C/D)을 별도 필드로 정의, explanation 누락 |
