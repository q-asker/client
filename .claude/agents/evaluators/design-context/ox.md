# OX(참/거짓) 설계 컨텍스트

> **이 파일은 OX(참/거짓) 프롬프트 평가 시에만 참조합니다.**
> 공통 설계 컨텍스트(`common.md`)와 함께 사용하세요.

---

## 1. 형성 평가 단일 모드

OX 퀴즈는 **형성 평가 전용**으로 설계되었습니다. 분기 로직이 없습니다.

| 설계 결정   | 내용                                                                   |
| ----------- | ---------------------------------------------------------------------- |
| 평가 목적   | 형성 평가 — 핵심 개념의 정확한 이해 여부를 참/거짓 판단으로 확인       |
| 인지 수준   | Bloom's Taxonomy '기억(Remember)' + '이해(Understand)'                 |
| 예상 정답률 | 설계 의도에 따라 다름 — 단순 사실 확인(85~90%), 미세 변조 판별(65~75%) |

**평가 시 주의**: `평가 목적` placeholder나 형성/총괄/진단 분기 로직을 제안하거나 도입하는 것은 금지입니다.

## 2. 고정값 요소

아래 요소는 설계자가 확정한 고정값이며, 변경/파라미터화를 제안하지 마세요:

- **선택지**: O(참) / X(거짓) 2개 고정
- **O/X 비율**: 전체 문항에서 O와 X가 균등 배분 (편향 금지)
- **진술문 규칙**: 단일 개념/사실만 판단하는 명확한 서술문, 복합 조건 금지
- **거짓 진술 변조 전략**: 3유형(사실 변조형·관계 변조형·조건 변조형) 중 다양하게 활용
- **해설 구조**: 단일 계층 — [판정] + [근거] + [교정 진술](거짓인 경우) + [학습 포인트]
- **난이도 배분**: 사실 확인(기억) / 이해 판별(이해) 균등
- **대상 학습자**: 대학 학부 수준
- **셔플**: O/X 선택지는 순서가 고정(O 먼저, X 다음)이므로 셔플 불필요

## 3. OX 퀴즈 핵심 설계 원칙

### 3.1 진술문 설계 전략

OX 퀴즈의 변별력은 **진술문 자체의 설계 품질**에 의존합니다. 4지선다와 달리 오답 선택지가 없으므로, 거짓 진술의 변조 방식이 핵심입니다.

#### 거짓 진술 변조 3유형

| 유형        | 설명                                     | 예시                                                 | 진단 목표               |
| ----------- | ---------------------------------------- | ---------------------------------------------------- | ----------------------- |
| 사실 변조형 | 핵심 용어/수치/명칭을 유사한 것으로 교체 | "TCP는 비연결형 프로토콜이다" (연결형→비연결형)      | 정확한 사실 기억        |
| 관계 변조형 | 개념 간 관계(인과, 포함, 순서)를 뒤바꿈  | "상속은 캡슐화의 하위 개념이다" (관계 오류)          | 개념 간 관계 이해       |
| 조건 변조형 | 적용 조건/범위/전제를 미세하게 변경      | "모든 객체지향 언어는 다중 상속을 지원한다" (일부만) | 조건/범위의 정확한 이해 |

#### 진술문 함정 회피 규칙

| 금지 항목        | 이유                                        | 나쁜 예                          |
| ---------------- | ------------------------------------------- | -------------------------------- |
| 이중 부정        | 판단 혼란 유발, 언어 능력 측정됨            | "~하지 않는 것은 아니다"         |
| 절대 표현 남용   | "항상", "절대로"는 정답 단서가 됨           | "항상 ~한다" → 학습자가 X를 추측 |
| 복합 진술        | 두 사실을 동시에 판단하면 부분 참/거짓 발생 | "A는 B이고, C는 D이다"           |
| 사소한 세부사항  | 핵심 개념이 아닌 지엽적 내용                | 특정 버전 번호, 날짜 등          |
| 의견/주관적 판단 | 객관적 참/거짓 판정 불가                    | "Python은 가장 좋은 언어이다"    |

### 3.2 참(O) 진술문 설계 주의사항

- 참 진술은 **강의노트 원문에 가까운 표현**으로 작성하되, 단순 복사가 아닌 핵심 의미 보존
- 지나치게 쉬운 참 진술(자명한 사실)은 변별력이 없으므로 핵심 개념 중심으로 선별
- 참 진술도 학습자가 "정말 맞나?" 하고 한 번 생각하게 만드는 수준이 적절

## 4. 출력 구조

JSON 스키마 의존성을 최소화하기 위해, 프롬프트에서는 **3개 핵심 필드**만 명시합니다:

- `content` (판단 대상 진술문)
- `selections` (O/X 선택지 — 각 `content`("O" 또는 "X") + `correct` boolean + `explanation`)
- 정답 선택지의 `explanation`에 전체 해설 통합

**MCQ/빈칸과의 차이**: MCQ는 문제 본문 + 4개 선택지, 빈칸은 빈칸 서술문 + 4개 선택지이지만, OX는 진술문 + O/X 2개 선택지입니다. 해설의 핵심은 "왜 참/거짓인가"의 근거와 거짓인 경우 올바른 진술로의 교정입니다.

**평가 시 주의**: 출력 필드를 세분화하거나 추가 필드를 제안하지 마세요. 스키마는 별도로 관리됩니다.

## 5. OX 설계 준수 평가 체크리스트

> 아래 체크리스트는 OX 프롬프트 평가 시에만 적용합니다.
> 공통 체크리스트(`common.md` §4)와 함께 사용하세요.

### 형성 평가 모드 준수 → `clarity`, `difficulty_control` 반영

| #   | 체크 항목                                               | 위반 예시                                                |
| --- | ------------------------------------------------------- | -------------------------------------------------------- |
| O1  | 평가 목적이 형성 평가 고정인가? placeholder나 분기 없음 | `{평가 목적 삽입 위치}`, `총괄 평가인 경우...` 분기 존재 |
| O2  | 인지 수준이 Bloom's '기억+이해' 범위인가?               | 적용/분석/평가 수준 문항 포함                            |
| O3  | 대상 학습자가 "대학 학부 수준"으로 고정인가?            | `{학과/학년 삽입 위치}` placeholder 존재                 |

### 진술문 설계 규칙 준수 → `clarity`, `answer_quality` 반영

| #   | 체크 항목                                                                | 위반 예시                                  |
| --- | ------------------------------------------------------------------------ | ------------------------------------------ |
| O4  | 진술문이 단일 개념/사실만 판단하도록 규칙이 있는가? (복합 진술 금지)     | 복합 진술 허용, 부분 참/거짓 가능          |
| O5  | 거짓 진술 변조 전략(3유형 이상)이 정의되었는가?                          | 변조 유형 미정의, 단순 부정만 사용         |
| O6  | 함정 회피 규칙(이중 부정, 절대 표현, 사소한 세부사항 등)이 명시되었는가? | 함정 회피 규칙 없음                        |
| O7  | 참(O) 진술의 설계 기준이 있는가? (자명한 사실 회피, 핵심 개념 중심)      | 참 진술 설계 기준 없음, 자명한 사실만 출제 |

### 고정값 준수 → `difficulty_control`, `answer_quality` 반영

| #   | 체크 항목                                                                 | 위반 예시                             |
| --- | ------------------------------------------------------------------------- | ------------------------------------- |
| O8  | 선택지가 O/X 2개 고정인가?                                                | "O/X/모름" 3개, 또는 선택지 수 가변   |
| O9  | O/X 비율 균등 배분 규칙이 명시되었는가?                                   | 비율 규칙 없음, 또는 한쪽에 편향 허용 |
| O10 | 난이도 배분(사실 확인/이해 판별 균등)이 명시되었는가?                     | 난이도 배분 없음, 사실 확인에만 편중  |
| O11 | 거짓 진술에서 변조할 핵심 요소를 강의노트 기반으로 결정하도록 지시하는가? | 외부 지식 기반 변조 허용              |

### 해설 구조 준수 → `answer_quality`, `model_optimization` 반영

| #   | 체크 항목                                                                    | 위반 예시                               |
| --- | ---------------------------------------------------------------------------- | --------------------------------------- |
| O12 | 해설에 [판정] + [근거] + [교정 진술](거짓 시) + [학습 포인트] 구조가 있는가? | 구조 미정의, "맞습니다/틀립니다"만 제시 |
| O13 | 거짓 진술 해설에 "올바른 진술"로의 교정이 포함되는가?                        | 거짓이라고만 하고 올바른 내용 미제시    |
| O14 | 해설에 강의노트 출처 직접 인용(`> 근거: [섹션명] > "인용"`) 규칙이 있는가?   | 출처 인용 없이 설명만 제공              |
| O15 | 참 진술 해설에도 "왜 참인가"의 근거가 포함되는가? (단순 "맞습니다" 금지)     | 참 진술에 대한 해설이 형식적            |

### 출력 구조 준수 → `model_optimization` 반영

| #   | 체크 항목                                                                       | 위반 예시                                  |
| --- | ------------------------------------------------------------------------------- | ------------------------------------------ |
| O16 | 출력 필드를 `content`, `selections` 중심으로 안내하는가?                        | 13개 이상 상세 필드 정의, JSON 스키마 중복 |
| O17 | selections이 O/X 2개로 구성되고 각각 `content` + `correct` + `explanation`인가? | 선택지 구조 미정의, explanation 누락       |

## 6. 다른 유형과의 주요 차이점 (평가 참고)

| 항목           | MCQ(4지선다)                   | 빈칸 채우기                     | OX(참/거짓)                       |
| -------------- | ------------------------------ | ------------------------------- | --------------------------------- |
| 인지 수준      | Bloom's 적용~분석 (중/상 균등) | Bloom's 기억 (단일)             | Bloom's 기억+이해                 |
| 평가 목적      | 통합 모드 (형성+총괄+진단)     | 형성 평가 전용                  | 형성 평가 전용                    |
| 선택지 수      | 4개                            | 4개 (정답 1 + 오답 3)           | 2개 (O/X)                         |
| 문제 구조      | content (문제 본문)            | title (빈칸 포함 서술문)        | content (판단 대상 진술문)        |
| 변별력 원천    | 오답 선택지의 매력도           | 빈칸 문맥의 정답 유일성         | 진술문 변조의 정교함              |
| 오답/변조 전략 | 조건 누락·오개념·무관 사실형   | 유사 개념·혼동 유발·무관 개념형 | 사실 변조·관계 변조·조건 변조형   |
| 해설 핵심      | 오답별 교정 + 심화 학습        | 정답 근거 + 오답별 구별 포인트  | 참/거짓 판정 근거 + 교정 진술     |
| 정답 비율 관리 | 서버 셔플 (프롬프트 미지시)    | 서버 셔플 (프롬프트 미지시)     | O/X 균등 배분 (프롬프트에서 지시) |
| 난이도 배분    | 중/상 균등                     | 없음 (단일 수준)                | 기억 / 이해 균등                  |
