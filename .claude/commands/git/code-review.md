# GitHub PR 코드 리뷰 커맨드

GitHub MCP를 사용하여 열린 PR을 선택하고, 코멘트 단위로 코드 리뷰를 수행하는 커맨드.
각 코멘트는 GitHub PR 코멘트로 그대로 붙여넣을 수 있는 수준의 구체성을 갖춘다.

---

## 0단계: GitHub MCP 가용성 확인

ToolSearch로 GitHub 관련 MCP 도구를 탐색한다. 도구명은 환경마다 다를 수 있으므로 반드시 동적으로 검색한다.

```
ToolSearch: "+github" 또는 "github pull request" 등으로 검색
```

**MCP를 찾지 못한 경우** — 아래 안내를 출력하고 중단:

```
⚠️ GitHub MCP가 설정되지 않았습니다.

GitHub MCP 설정 방법:
1. Claude Code 설정에서 MCP 서버 추가
2. GitHub Personal Access Token 발급 (repo 스코프 필요)
3. MCP 서버 설정에 토큰 등록

설정 완료 후 다시 /code-review를 실행해주세요.
```

---

## 1단계: 리포지토리 정보 수집

```bash
git remote -v
```

출력에서 `owner/repo` 형태의 리포지토리 정보를 추출한다.

---

## 2단계: PR 목록 표시 & 선택

GitHub MCP로 열린(open) PR 목록을 조회하고 아래 형태로 표시한다:

```
📋 열린 PR 목록:

 번호  | 제목                           | 작성자
-------|-------------------------------|--------
 #142  | 퀴즈 생성 로직 리팩토링          | username
 #141  | SSE 연결 안정화                 | username
 #139  | 로그인 플로우 개선               | username

리뷰할 PR 번호를 입력해주세요:
```

AskUserQuestion으로 사용자에게 PR 번호를 입력받는다.

---

## 3단계: PR diff 수집 & 프로젝트 규칙 파악

**병렬로 수행:**

1. **PR 정보 수집** (GitHub MCP):
   - PR의 changed files 목록 + diff
   - PR body (설명)
   - head/base 브랜치 정보

2. **프로젝트 규칙 파악**:
   - `CLAUDE.md` — 기술 스택, 아키텍처, 컨벤션
   - `.claude/rules/code-rules.md` — 코딩 규칙, FSD 구조, import 규칙, 제약 사항
   - 기타 `.claude/rules/` 파일 중 코드 품질 관련 규칙

이 정보를 기반으로 리뷰 기준을 구성한다. **프로젝트 규칙은 동적으로 읽어서 적용하며, 특정 프레임워크/라이브러리를 이 커맨드에 하드코딩하지 않는다.**

---

## 4단계: 코멘트 단위 리뷰 + Q&A 반복

### 리뷰 기준

- **공통**: 로직 오류, 엣지 케이스, 예외 처리, 성능, 보안, 가독성, 유지보수성
- **프로젝트 규칙**: 3단계에서 파악한 CLAUDE.md 기술 스택·아키텍처 + code-rules.md 컨벤션 적용

### 리뷰 진행 방식

diff에서 리뷰 포인트를 식별하고, **각 코멘트에 대해 아래 사이클을 반복**한다:

1. **리뷰 초안 제시** — 파일명 + 라인 범위 + 발견 사항 + 심각도
2. **개발자에게 질문** (AskUserQuestion) — 의도, 이유, 대안 고려 여부 등
3. **답변 반영** — 개발자의 답변을 반영하여 코멘트를 구체화하거나 철회
4. **코멘트 확정** — 최종 확정된 코멘트를 내부 목록에 추가

### 코멘트 제시 형식

```
---
📝 코멘트 {N}/{예상 총 수}

**파일**: `src/path/to/file.js`
**라인**: L42-L55
**심각도**: 🚨 높음 / ⚠️ 중간 / 💡 낮음

{리뷰 내용 초안}

💬 질문: {개발자에게 확인할 사항}
---
```

### 대규모 diff 처리

변경 파일이 많을 경우 파일별 또는 기능 단위로 분할하여 진행한다. 분할 시 사용자에게 진행 상황을 알린다:

```
📂 파일 그룹 1/3: 퀴즈 생성 관련 (3개 파일)
- src/features/quiz-generation/model/useQuizGeneration.js
- src/features/quiz-generation/ui/QuizForm.jsx
- src/shared/api/quizApi.js
```

## 5단계: 리뷰 문서 저장

모든 코멘트가 확정되면 `docs/PR-{number}-review.md` 파일로 저장한다.

### 문서 구조

```markdown
# PR #{번호} 코드 리뷰: {제목}

**브랜치**: `{head}` → `{base}`
**리뷰 일시**: {YYYY-MM-DD}
**변경 파일**: {N}개

---

**라인**: L42-L55

{리뷰 내용 — GitHub PR 코멘트로 그대로 붙여넣을 수 있는 수준}

---
```

## 범위 제한

- **코멘트 작성(GitHub에 게시)이나 PR 승인/반려는 범위 밖** — 리뷰 문서 생성까지만 수행
- 생성된 문서의 코멘트를 GitHub에 게시하는 것은 사용자가 직접 수행
