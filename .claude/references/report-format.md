# 초기화 리포트 공통 포맷

초기화 워크플로우(`/init` → `/sync-claude` → `/plan-feature`)의 완료 리포트 출력 규격.
각 커맨드는 이 포맷을 따르고, 본문만 자체 정의한다.

## 워크플로우 정의

| 단계 | 커맨드          | 니모닉 |
| ---- | --------------- | ------ |
| 1    | `/init`         | 세팅   |
| 2    | `/sync-claude`  | 동기화 |
| 3    | `/plan-feature` | 계획   |

## 출력 구조

```
---

## 프로젝트 초기화 [{진행 바}] {현재 단계}/3 완료

> **세팅 → 동기화 → 계획**

{각 커맨드의 고유 본문}

### 초기화 진행 상황

| 단계 | 커맨드 | 상태 |
|---|---|---|
| 1 | `/init` | {상태} |
| 2 | `/sync-claude` | {상태} |
| 3 | `/plan-feature` | {상태} |

{조건부: Shrimp 안내}

---

> 다음: {다음 커맨드}
```

## 진행 바 규칙

| 완료 단계 | 진행 바       |
| --------- | ------------- |
| 1/3       | `[███░░░░░░]` |
| 2/3       | `[██████░░░]` |
| 3/3       | `[█████████]` |

## 상태 결정 규칙

**현재 단계**: `✅ 완료`
**이전 단계**: `✅ 완료`
**다음 단계**: 아래 조건에 따라 결정

### 2단계 `/sync-claude` 상태

- `/init` 직후라면 → `⚠️ 동기화 필요 → 반드시 실행`
- 이미 동기화된 프로젝트라면 → `✅ 동기화 완료 → 건너뛰기 가능`

### 3단계 `/plan-feature` 상태

- `docs/PRD.md` 없음 → `⚠️ PRD 없음 — 반드시 실행`
- `docs/PRD.md` 있고 `docs/roadmaps/` 없음 → `⚠️ ROADMAP 없음`
- 둘 다 존재 → `✅ PRD+ROADMAP 존재 — 건너뛰기 가능`

## 다음 단계 안내

- 다음 미완료 커맨드가 있으면 → `> 다음: {커맨드}`
- 모든 단계 완료 → `> 초기화 완료! 이제 개발을 시작하세요.`

## 조건부 블록

- `.shrimp/` 디렉토리가 프로젝트 루트에 존재하지 않을 때만:
  `💡 **Shrimp Task Manager**를 설치하면 체계적 작업 관리가 가능합니다. (선택)`

## 출력 규칙

- 프로그레스 바는 리포트 최상단에 배치
- 니모닉은 프로그레스 바 바로 아래에 배치
- 본문 아래에 초기화 진행 상황 테이블 배치
- 리포트 마지막은 반드시 `> 다음:` 또는 완료 메시지로 종료
